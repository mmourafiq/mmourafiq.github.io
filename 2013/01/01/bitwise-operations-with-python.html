Bitwise operations with python.

Handling string of bits.

Bitwise operators allows us to write more efficient code due to the fact that CPUs are good at handling bits. They treat numbers naturally but in different way, indeed, instead of considering a number as single value they treat it as a string of bits.

  • 0 is written as “0”
  • 1 is written as “1”
  • 2 is written as “10”
  • 3 is “11”
  • …

Important bitwise operators to know:

  • x « y : Returns x with the bits shifted to the left by y places (and new bits on the right-hand-side are zeros). This is the same as multiplying x by 2**y.
  • x » y : Returns x with the bits shifted to the right by y places. This is the same as dividing x by 2**y.
  • x & y : Does a “bitwise and”. Each bit of the output is 1 if the corresponding bit of x AND of y is 1, otherwise it’s 0.
  • x | y : Does a “bitwise or”. Each bit of the output is 0 if the corresponding bit of x AND of y is 0, otherwise it’s 1.
  • ~ x : Returns the complement of x - the number you get by switching each 1 for a 0 and each 0 for a 1. This is the same as -x - 1.
  • x ^ y : Does a “bitwise exclusive or”. Each bit of the output is the same as the corresponding bit in x if that bit in y is 0, and it’s the complement of the bit in x if that bit in y is 1.

Now let’s do some experiments with these bitwise operators.

Finding even and odd numbers:

 1 #odd or even the way most people would do
 2 odd_or_even = lambda n: "Even" if n%2==0 else "Odd"
 3 #odd or even using bitwise operations
 4 odd_or_even = lambda n: "Even" if n&1==0 else "Odd"
 5 #the last odd_or_even function has the virtue to only check the last digit.
 6 #Conversely, the former deals with division, and it would be annoying to do long division.
 7 #The "&" operator performs a bitwise AND.
 8 #If our number was 17 then it would yield 1 because only the first binary column for both numbers contains a 1.
 9 #Likewise if the number was 60 then we'd get back 0.
10 bin(17)
11 '0b10001'
12 odd_or_even(17)
13 'Odd'
14 
15 bin(60)
16 '0b111100'
17 odd_or_even(60)
18 'Even'

Double and half:

1 #double and half the intuitive way
2 double = lambda n: n*2
3 half = lambda n: n/2
4 #double and half using bitwise operations
5 double = lambda n: n << 1
6 half = lambda n: n >> 1

URL shortener:

Now we are going to see a more elaborated example of using bitwise operations; a URL shortener. I think you are more used to this concept since you probably use it everyday, either to access a post blog using a short URL, or as mean to shorten your URL and share it more conveniently.

In order for us to make this URL shortener, we will be in need of an object that, for instance, take the URL id from your database and generate from it a code. The same code will be used to retrieve the URL afterward.

 1 DEFAULT_ALPHABET = 'az7er5tyu1io0pq4sd9fg6hjk8lmw3xcv2bn'
 2 LEGTH_GENERATION = 32
 3 MIN_LENGTH = 6
 4 
 5 class URLEncoder(object):
 6     """
 7 It generates 36**6 = 2176782336 values (6 lowercase letters) which falls between 2**31 = 2147483648 and 2**32 = 4294967296.
 8 It pads the codes that are shorter than 6 characters with leading 'a' characters,
 9 """
10     def __init__(self, alphabet=DEFAULT_ALPHABET, length_generation=LEGTH_GENERATION):
11         self.alphabet = alphabet
12         self.length_generation = length_generation
13         self.mask = (1 << length_generation) - 1
14         self.mapping = range(length_generation)
15         self.mapping.reverse()
16 
17     def encode_url(self, n, min_length=MIN_LENGTH):
18         return self.__enbase(self.__encode(n), min_length)
19 
20     def decode_url(self, n):
21         return self.__decode(self.__debase(n))
22 
23     def __encode(self, n):
24         result = 0
25         for i, b in enumerate(self.mapping):
26             if (n & self.mask) & (1 << i):
27                 result |= (1 << b)
28         return (n & ~self.mask) | result
29 
30     def __decode(self, n):
31         result = 0
32         for i, b in enumerate(self.mapping):
33             if (n & self.mask) & (1 << b):
34                 result |= (1 << i)
35         return (n & ~self.mask) | result
36 
37     def __enbase(self, x, min_length=MIN_LENGTH):
38         result = self.__enbase_iter(x)
39         padding = self.alphabet[0] * (min_length - len(result))
40         return '%s%s' % (padding, result)
41 
42     def __enbase_iter(self, x):
43         n = len(self.alphabet)
44         if x < n:
45             return self.alphabet[x]
46         return self.__enbase_iter(x / n) + self.alphabet[x % n]
47 
48     def __debase(self, x):
49         n = len(self.alphabet)
50         result = 0
51         for i, c in enumerate(reversed(x)):
52             result += self.alphabet.index(c) * (n ** i)
53         return result
